Site E-commerce en Symfony

Description du Projet

Ce projet est un site e-commerce développé dans le cadre d'un projet académique utilisant le framework Symfony. L'objectif était de créer une plateforme de commerce électronique complète avec une gestion des produits, des utilisateurs et des commandes.

Fonctionnalités

Authentification des utilisateurs

Inscription, connexion et gestion des profils utilisateurs.

Catalogue produits

Affichage et gestion du catalogue de produits.

Édition produit

Interface d'édition et de gestion des produits.

Panier d'achat

Gestion du panier et processus de commande.

Base de Données

Base de données SQLite

La base de données utilise une structure complète avec plusieurs tables interconnectées :

Structure des tables principales
Table category
  • id (INTEGER, PRIMARY KEY, AUTOINCREMENT)
  • name (VARCHAR)
Table product
  • id (INTEGER, PRIMARY KEY, AUTOINCREMENT)
  • name (VARCHAR)
  • price (FLOAT/DECIMAL)
  • quantity (INTEGER)
  • category_id (INTEGER, FOREIGN KEY vers category.id)
Table user
  • id (INTEGER, PRIMARY KEY, AUTOINCREMENT)
  • email (VARCHAR, UNIQUE)
  • password (VARCHAR)
  • roles (TEXT/JSON)
Tables système Symfony
Table doctrine_migration_versions
  • version (VARCHAR, PRIMARY KEY)
  • executed_at (DATETIME)
  • execution_time (INTEGER)
Table messenger_messages
  • id (INTEGER, PRIMARY KEY, AUTOINCREMENT)
  • body (TEXT)
  • headers (TEXT)
  • queue_name (VARCHAR)
  • created_at (DATETIME)
  • available_at (DATETIME)
  • delivered_at (DATETIME)
Contraintes et relations
  • product.category_id est une clé étrangère vers category.id
  • Un produit appartient à une seule catégorie
  • Une catégorie peut avoir plusieurs produits
  • La table user gère les utilisateurs (authentification)
  • Les tables doctrine_migration_versions et messenger_messages sont utilisées par Symfony pour la gestion des migrations et des messages asynchrones
